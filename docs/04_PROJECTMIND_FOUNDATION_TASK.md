 ProjectMind Foundation v0.1 — первое настоящее задание на создание приложения

Ты разрабатываешь первую рабочую версию приложения ProjectMind.

Это не прототип ради красоты. Это рабочий инструмент, который будет использоваться для первого реального клиента — психолога Марины Дугиной.

В проекте уже есть документация в папке docs. Важный документ: docs/03_MVP_SPECIFICATION.md. Используй его как основу MVP, но не уходи в лишние функции. MVP уже описан как система диагностики проекта, сохранения ответов, AI-анализа и подготовки основы для лендинга/рекламы.

Главная задача

Создать первую рабочую версию веб-приложения ProjectMind, в которой маркетолог может:

открыть приложение;
создать рабочее пространство клиента;
заполнить или отправить клиенту диагностический опросник;
сохранить ответы;
увидеть структурированную карточку проекта;
подготовить данные для дальнейшего AI-анализа.

В этой версии AI можно пока сделать как подготовленный экран/заглушку без подключения OpenAI. Главное — чтобы сама система, интерфейс, структура проекта и диагностика уже работали.

1. Важные правила разработки

Перед началом работы:

Изучи все файлы в папке docs.
Не создавай лишние функции.
Не делай CRM, оплату, биллинг, рекламный кабинет, рассылки, сложную аналитику.
Не усложняй архитектуру.
Не задавай мне вопросы, если можешь принять разумное решение внутри MVP.
Сделай первую версию максимально понятной для обычного человека.
Интерфейс должен быть на русском языке.
Слова должны быть простыми, человеческими, без перегруза профессиональным жаргоном.
Приложение должно быть аккуратным, современным, спокойным, светлым и удобным.
После выполнения обязательно напиши, что создано, какие файлы изменены и как запустить проект.

Цель — быстро получить рабочую локальную версию.

3. Название продукта

Название продукта:

ProjectMind

Подзаголовок:

Система ведения клиентских проектов от диагностики до запуска

Главная идея:

ProjectMind помогает специалисту не начинать работу вслепую, а сначала собрать всю информацию о клиенте, понять цель проекта, целевую аудиторию, сильные стороны, ограничения и только после этого переходить к лендингу и рекламе.

4. Основная логика приложения

Приложение должно работать как рабочий кабинет маркетолога.

Главная сущность — рабочее пространство клиента.

Примеры рабочих пространств:

Марина Дугина
Сайт стилиста
Онлайн-курс
Новый лендинг
Продвижение консультации

Каждое рабочее пространство содержит:

общую информацию;
цель проекта;
диагностику;
ответы клиента;
этапы работы;
будущий AI-анализ.

6. Диагностика проекта

Это самый важный экран MVP.

Диагностика должна быть удобной, человеческой и не похожей на сухую анкету.

Название:

Стратегическая диагностика проекта

Вступительный текст:

Эта диагностика помогает глубоко понять проект до создания лендинга и запуска рекламы. Чем подробнее будут ответы, тем точнее получится стратегия, оффер и дальнейшие материалы.

Заполнение можно проходить спокойно, по шагам. Ответы сохраняются автоматически.

После завершения диагностики статус проекта меняется на «Диагностика завершена».
В карточке проекта появляется кнопка «Подготовить AI-анализ».
В MVP кнопка «Подготовить AI-анализ» открывает экран-заглушку с текстом:
«AI-анализ будет подключен на следующем этапе. Сейчас вы можете просмотреть ответы клиента и использовать их для подготовки стратегии, лендинга и рекламы».
8. Экран «Ответы клиента»

Экран должен содержать:

название блока;

Ответ не заполнен.

10. Дизайн и UX

Стиль:

светлый;
спокойный;
современный;
чистый;
без перегруза;
с хорошими отступами;
крупные читаемые заголовки;
мягкие карточки;
понятные кнопки;
аккуратные статусы.

Цветовая идея:

фон: очень светлый теплый серый или белый;
карточки: белые;
основной акцент: глубокий синий, графитовый или фиолетово-синий;
текст: темно-графитовый;
вторичный текст: серый.

Не использовать агрессивные цвета.

Интерфейс должен быть понятен человеку без технической подготовки.

11. Структура данных в LocalStorage

Создай хранение проектов в LocalStorage.

Примерная структура проекта:

type Workspace = {
  id: string;
  title: string;
  clientName: string;
  niche: string;
  projectFormat: string;
  mainGoal: string;
  comment?: string;
  status: 'draft' | 'diagnostic_started' | 'diagnostic_completed' | 'ready_for_analysis' | 'in_progress';
  createdAt: string;
  updatedAt: string;
  diagnosticProgress: number;
  answers: DiagnosticAnswer[];
};

type DiagnosticAnswer = {
  blockId: string;
  questionId: string;
  questionText: string;
  answer: string | string[];
  updatedAt: string;
};

Можно улучшить структуру, если это разумно, но не усложнять.

13. Маршруты приложения

Сделай маршруты:

/ — стартовая страница
/workspaces — список рабочих пространств
/workspaces/new — создание рабочего пространства
/workspaces/:id — карточка рабочего пространства
/workspaces/:id/diagnostic — диагностика
/workspaces/:id/answers — ответы клиента
/workspaces/:id/ai-analysis — AI-анализ

Можно использовать React Router.

Если React Router не установлен — установи его.

14. Проверка качества

После разработки проверь:

Приложение запускается.
Можно создать рабочее пространство.
Оно появляется в списке.
Можно открыть карточку.
Можно пройти диагностику.
Ответы сохраняются.
Прогресс меняется.
После завершения статус меняется.
Можно открыть ответы.
Можно открыть AI-анализ.
После перезагрузки страницы данные не исчезают.
Интерфейс адаптивен для ноутбука и телефона.
