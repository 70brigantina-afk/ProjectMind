Задание для Cursor

Мы начинаем разработку большого коммерческого AI-продукта.

Очень важно соблюдать следующие правила.

Правила проекта
Никогда не создавать большой объем кода за один запрос.
Перед началом каждого этапа полностью изучать всю документацию проекта.
Если документации недостаточно — не придумывать самостоятельно. Вместо этого сформулировать список вопросов.
Никогда не удалять существующий функционал без явного указания.
Никогда не переписывать работающий код полностью, если можно расширить существующую архитектуру.
Использовать современную модульную архитектуру.
Соблюдать принципы SOLID, DRY, KISS и Clean Architecture.
Каждый модуль должен быть независимым.
Каждый компонент должен иметь понятное название.
После завершения каждого этапа необходимо:
проверить код;
выполнить самопроверку;
предложить улучшения;
дождаться подтверждения пользователя.
Перед написанием кода сначала составлять план реализации.
После прочтения каждого нового документа подтверждать понимание требований.
Если обнаружены противоречия в документации, остановиться и сообщить об этом.
Не использовать устаревшие технологии.
Использовать лучшие современные практики разработки.

Сейчас ничего не программируй.

Только изучи структуру проекта, подтверди понимание правил и предложи оптимальную архитектуру проекта высокого уровня.

После этого остановись и жди дальнейших инструкций.
